# Phase 1 Data Model: Game Rules Section

**Not applicable.**

This feature displays fixed content only. There are no entities, no database columns, no persisted or fetched data, and no state transitions. The rules are hard-coded constants inside the `RulesSection` component (see [research.md](./research.md), Decision 1).

If a future feature needs editable rules, that will introduce a data model at that point — deliberately out of scope here per Constitution III (YAGNI).
